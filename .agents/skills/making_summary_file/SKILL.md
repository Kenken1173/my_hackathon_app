# 解説内容のまとめと記録

## このスキルの目的
- 解説してもらった内容を記録し、次回は自分の力で実装できる状態にする。
- 毎回出力先やフォーマットを指示しなくても、安定した形式で追記される。

## いつ使うか
- チャット上で解説や質疑応答を行った **後に**、ユーザーが「まとめて」と指示したとき。
- 解説の開始から「まとめて」の指示までの間にあった全てのやり取り（追加の質問・補足説明を含む）を対象にする。

## 出力先
- `study_after_make_project/learnings_and_insights.md`
- 既存の内容は消さず、末尾に追記する。

## 手順

### 1. 対象範囲の特定
- 今回の会話で解説した内容を振り返る。
- 追加の質問やそれに対する回答も含める。
- 解説の主題が何だったかを特定する。

### 2. 出力先ファイルの確認
- `study_after_make_project/learnings_and_insights.md` を読み、現在の内容を把握する。
- 重複する内容がないか確認する。

### 3. 追記内容の構成
以下のフォーマットで追記する。

```markdown
## [トピック名]

### 調べ方・考え方の手順
- このトピックを自力で調べるとき、どこから見ればよいか。
- 確認すべきファイルや設定の順番。

### このプロジェクトで分かったこと
- 調査の結果、実際に確認できた事実。
- テーブル構成、設定値、コードの動きなど具体的な内容。

### 注意点・改善のヒント
- 現状の設計で気をつけるべき点。
- 今後改善できそうな箇所。

### 一言まとめ
- このトピックを一文で要約する。
```

### 4. 追記の実行
- `study_after_make_project/learnings_and_insights.md` の末尾に追記する。
- 既存の内容はそのまま残す。

### 5. 完了報告
- 追記した内容の概要をチャット上で簡潔に報告する。

## 守るべきルール
- 解説の開始から「まとめて」までの **全てのやり取り** を反映する。途中の追加質問や補足も漏らさない。
- 「調べ方・考え方の手順」を必ず含める。これが次回自力でやるための最重要パートである。
- このプロジェクト固有の事実と、一般的な知識を区別して書く。
- 冗長にならないよう箇条書きを基本とする。
- 既存の記載と重複する場合は、新しい知見の差分だけを追記する。
